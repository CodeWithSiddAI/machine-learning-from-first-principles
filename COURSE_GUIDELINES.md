# COURSE_GUIDELINES.md

# Course Standards

These guidelines define how every lesson in this course must be created.

---

# Lesson Template

Every lesson must include the following sections in order.

1. Learning Objectives
2. Story-based Introduction
3. Intuition
4. Visual Explanation (ASCII diagrams when useful)
5. Feynman Explanation
6. Mathematical Foundations
7. Worked Examples
8. Python Implementation
9. Common Mistakes
10. Practice Problems
11. Coding Assignment
12. Interview Questions
13. Reflection
14. Summary

No lesson should skip these sections unless the topic genuinely does not require them.

---

# Explanation Standards

Every difficult concept should first answer:

* Why was this invented?
* What problem does it solve?
* What would happen if it didn't exist?
* Where is it used in practice?

Only after these questions are answered should formulas be introduced.

---

# Mathematics Rules

Every equation must explain:

* Each variable
* Each constant
* Units (when applicable)
* Why the equation works
* How it was derived (when appropriate)

Students should understand equations instead of memorizing them.

---

# Coding Rules

Use:

* Python
* NumPy

first.

Introduce:

* Pandas
* Matplotlib
* Scikit-Learn
* PyTorch

only when they naturally fit the curriculum.

Every important line of code should be explained.

Readable code is preferred over clever code.

---

# Assignments

Every lesson should include:

* Easy questions
* Medium questions
* Challenge questions
* One coding task
* One conceptual task

Assignments should reinforce understanding rather than memorization.

---

# Weekly Structure

Monday–Saturday

New concepts.

Sunday

No major new concepts.

Instead:

* Revision
* Weekly Project
* Quiz
* Reflection
* Interview Questions

---

# Writing Style

Write like an experienced university professor.

Friendly.

Patient.

Encouraging.

Use stories.

Use analogies.

Use real-world examples.

Use diagrams whenever possible.

Avoid unnecessary complexity.

---

# Quality Standard

Every lesson should be detailed enough that a student can learn the topic without searching for another resource.

External resources may be optional for further reading but should never be required to understand the lesson.

This course should feel like a complete textbook rather than a collection of notes.
